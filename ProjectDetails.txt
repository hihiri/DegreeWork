Documentation OverLeaf link: https://www.overleaf.com/project/6295c413cfd1dda357467b5d

c++ server and a python client.

Server does not need to support multiple clients.
They communicate via TPC/IP protocol.
Both Have a config, and a log.

Message trafic is loged on both sides if the corresponding config boolean is true.
Messages are ASCII/text encoded.
Exceot for sendData payload, which is binary serialized.

ConfigFormat: JSON
Config content:
"log":boolean
"port":int
"cfdConfig": object (width, height)

Server ip is in client config, wont be sent in putConfig

Messages:
General message format: dxxxxxxxxxxxxxxxxxxxx where d is a single digit rest is the message content, lenght depends on the message type

SendConfig:
Client sends the current config content to server.
Server on receive, logs that it is received. Server updates its own config with the values received.
message format: 0|log|width|height
Example: 0|1|4096|32
Parameters:
  log: Logging flag (0 or 1)
  width, height: Grid dimensions for CFD simulation

GetStatus:
Server on receiving sends back the current computation status as Response message, computation status for now is just an internal int default 0
message content: 1
response format: 5s where s is the status

SendData:
Client sends CFD input payload from a file to Server.
Server on receiving:
If the status was not 0, sends back error response and does nothing else.
Otherwise stores the binary payload and acknowledges receipt.
Client reads bytes from file in client/data/ directory.
message format: 2|payloadSize|<binary payload>
Example: 2|131072|<131072 raw bytes>
Parameters:
  payloadSize: Number of payload bytes to read after the second delimiter
  binary payload: Raw CFD input bytes for later FPGA buffer handoff
Worth to note that at this point, cfd_init.txt is just a placeholder. Has no relevant data.
error message content: 6
ack message content: 7

GetResult:
If the satus was not "done" sends back error response. (currently for debug purposes status 1 is considered "done")
Else sends back the result value. (Currently placeholder value is 2026)
message content: 3
error response: 8
response content: 9xxxxxxxxxxxxx where x is placeholder for result value
